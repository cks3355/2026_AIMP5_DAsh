# DAsh Product Requirements

관련 문서: [Agents](agents.md), [TRD](trd.md), [Skill Guide](skill.md)

## 제품 개요

DAsh는 데이터 표준 담당자와 개발자가 표준용어를 빠르게 조회, 추천, 등록, 수정, 삭제할 수 있도록 돕는 Streamlit 기반 업무 도구다. 자연어 설명을 입력하면 기존 표준용어와 비교하고, 표준단어 기반 후보를 추천하며, 검증된 변경 요청을 저장소에 반영한다.

## 사용자

- 데이터 아키텍트: 표준용어 품질을 검토하고 등록 기준을 관리한다.
- 데이터 모델러: 신규 컬럼 또는 속성에 맞는 표준용어를 찾거나 신청한다.
- 개발자: 업무 용어의 의미와 물리명을 빠르게 확인한다.

## 핵심 사용자 흐름

1. 사용자가 업무 설명을 입력한다.
2. 조회 버튼으로 유사 표준용어를 확인한다.
3. 추천 버튼으로 표준단어 조합 후보를 확인한다.
4. 후보를 선택해 등록 요청을 수행한다.
5. 변경 버튼으로 등록, 수정, 삭제 요청을 직접 처리한다.
6. Agent 처리 로그에서 routing과 tool 실행 흐름을 확인한다.

## 기능 요구사항

- 자연어 입력을 6개 intent로 분류한다: meaning_select, term_recommend, term_insert, term_update, term_delete, da_request.
- Agent별 처리 책임은 [Agents](agents.md)를 따른다.
- 추천과 검증 규칙은 [Skill Guide](skill.md)를 따른다.
- 저장소, graph, UI 구현 조건은 [TRD](trd.md)를 따른다.
- 처리 로그는 사용자가 routing과 tool 실행 결과를 추적할 수 있게 표시한다.
- v6 앱은 프로젝트 문서(agents/prd/trd/skill)를 사이드바 또는 CLI에서 확인할 수 있어야 한다.

## 비기능 요구사항

- 로컬 Mock 모드에서 외부 의존성 없이 실행 가능해야 한다.
- 운영 모드에서는 환경변수로 OpenAI, Oracle, Tavily 구성을 주입할 수 있어야 한다.
- UI는 넓은 업무 화면에서 조회, 추천, 변경 결과를 한 번에 비교할 수 있어야 한다.

## 성공 기준

- 표준용어 조회, 추천, 등록/수정/삭제 데모 시나리오가 CLI와 Streamlit에서 동작한다.
- 명세 문서와 코드의 Agent, skill, 기술 구성 설명이 서로 모순되지 않는다.
- 검증 실패 케이스가 변경 저장소에 반영되지 않는다.
